Caso 2 — chuva excessiva em trigo

O trigo é cultura de inverno, e o excesso de chuva — sobretudo perto da colheita — degrada a qualidade dos grãos e favorece doenças [40] (ver Agrometeorologia dos cultivos). Este caso espelha o anterior, mas com a lógica invertida: uma COP por chuva excessiva é plausível quando a precipitação observada ficou acima da climatologia (anomalia positiva). É implausível quando não houve excesso.

Lógica invertida do flag

O sinal da comparação inverte em relação à seca: agora a suspeita recai sobre anomalias não positivas (<= 0) — declarou-se excesso de chuva, mas choveu o normal ou menos:

#!/usr/bin/env python3
# Exemplo educacional — não operacional.
# Flag de plausibilidade para COPs de chuva excessiva em trigo.
#
# Lógica INVERTIDA em relação à seca: a suspeita recai sobre anomalias NÃO
# positivas (declarou excesso de chuva, mas não choveu mais que o normal).
# As anomalias (observado - climatologia, em mm) são dadas; positiva = mais
# chuvoso que o normal.

# ============================================================
# IMPORTS
# ============================================================

import pandas as pd


# ============================================================
# DADOS DE EXEMPLO (anomalias já calculadas, em mm)
# ============================================================

chuva_trigo = pd.DataFrame({
    'ref_bacen':          ['B1', 'B2', 'B3', 'B4', 'B5'],
    'anom_evento_mm':     [ 90.0, -20.0,  -5.0, 150.0,  10.0],
    'anom_acum_ciclo_mm': [220.0, -60.0,  40.0, 180.0, -30.0],
})


# ============================================================
# FLAGS DE PLAUSIBILIDADE (lógica invertida: anom <= 0 é suspeito)
# ============================================================

# Flag 1: sem excesso na janela do evento.
chuva_trigo['flag_chuva_sem_anom_evento'] = (
    chuva_trigo['anom_evento_mm'].notna() &
    (chuva_trigo['anom_evento_mm'] <= 0)
)

# Flag 2: sem excesso acumulado no ciclo.
chuva_trigo['flag_chuva_sem_anom_ciclo'] = (
    chuva_trigo['anom_acum_ciclo_mm'].notna() &
    (chuva_trigo['anom_acum_ciclo_mm'] <= 0)
)

# Flag combinado: declarou excesso de chuva, mas não houve excesso em nenhum recorte.
chuva_trigo['flag_chuva_implausivel'] = (
    chuva_trigo['flag_chuva_sem_anom_evento'] &
    chuva_trigo['flag_chuva_sem_anom_ciclo']
)


# ============================================================
# DIAGNÓSTICO
# ============================================================

n_total = len(chuva_trigo)
n_implausivel = int(chuva_trigo['flag_chuva_implausivel'].sum())

print(f'Total de COPs:              {n_total}')
print(f'Implausíveis (priorizar):   {n_implausivel}')
print('-' * 50)
print(chuva_trigo[['ref_bacen', 'anom_evento_mm', 'anom_acum_ciclo_mm',
                   'flag_chuva_implausivel']].to_string(index=False))

Fallback hierárquico INMET → MERGE

Este caso introduz um refinamento importante para a qualidade do dado. A estação INMET, por ser uma medida direta no solo, é preferível para a precipitação pontual; a grade MERGE entra como alternativa quando não há estação com cobertura temporal suficiente na janela. A estratégia é hierárquica e explícita (sem troca silenciosa de fonte): tenta-se primeiro a estação mais próxima com cobertura acima de um limiar; só então recorre-se ao MERGE no centroide da COP, registrando qual fonte foi usada em cada caso.

Essa rastreabilidade da fonte é parte da auditabilidade: ao priorizar uma COP, sabe-se se a anomalia veio de uma estação próxima (mais confiável) ou de uma estimativa em grade.

Leitura

Como na seca, flag_chuva_implausivel apenas ordena a investigação: marca as COPs em que a chuva excessiva declarada não aparece nos dados. Comparar MERGE e INMET decêndio a decêndio, para o caso selecionado, ajuda a separar divergência de dado de incompatibilidade real — e a decisão final continua sendo do perito.
